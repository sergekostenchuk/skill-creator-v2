# Case Study: UI Intelligence Skill Group

This is a sanitized case study. It describes the operating lessons without publishing private project context, local paths, screenshots, or raw third-party code.

## Starting Point

The goal was to create a group of skills that could help an agent prepare stronger UI/UX directions before Figma or code work.

The group included specialized roles:

- design-intelligence-orchestrator
- award-design-intelligence
- commercial-design-reference-scout
- visual-trend-scout
- asset-and-section-scout
- design-pattern-extractor
- design-direction-synthesizer
- design-originality-guard

## What The First Runs Revealed

The early version could name sources and produce directions, but it was still too shallow for high-confidence design work.

Problems found during real use:

- Award directory cards were not enough evidence.
- Hero screenshots were not enough to understand page systems.
- Motion and interaction effects were often invisible in static screenshots.
- Public code, font, and script metadata needed to be inspected as metadata, without copying source code.
- File existence was not acceptance; outputs needed review and grading.
- The group needed to learn from failed runs and feed lessons back into the skill contracts.

## Improvements Added

The group was upgraded with an Award Live-Site Intelligence Protocol:

- open the actual live destination site, not only the award listing
- capture multiple scroll states where possible
- attempt relevant interaction states
- record public DOM/CSS/font/script metadata
- extract reusable principles, not copied layouts or code
- write originality boundaries for each direction
- preserve screenshots and metadata as evidence artifacts

The lesson was then generalized back into Skill Creator V2:

- External Source Evidence Ladder
- External Source Skill Contract
- Run-to-Skill Feedback Loop
- Shallow Evidence Anti-Pattern checks
- Skill Group Observability Contract
- Completion vs Acceptance hardening
- Evidence-aware 10/10 rubric

## What It Gave

The resulting workflow became stricter and more useful:

- It separated inspiration from copying.
- It caught weak evidence before downstream design work.
- It made live-source research repeatable.
- It produced better design directions because patterns were extracted from real pages, not just galleries.
- It gave the skill group a feedback loop, so practical failures became improvements to the skill system.

The most important result was cultural: "we created files" stopped being enough. The group had to prove that it observed the right sources, extracted the right patterns, preserved originality, and recorded evidence.
